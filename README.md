# Adjective Ordering in Referential Communication

Code, data, and manuscript for:

> **Adjective Ordering in Referential Communication: Subjectivity, Discriminatory Strength, and Incremental Pragmatics**
> Hening Wang, Fabian Schlotterbeck, and Michael Franke
> University of Tübingen

We develop a Rational Speech Act (RSA) model that crosses two dimensions in a 2×2
comparison: speaker architecture (global vs. incremental) and semantic regime
(context-fixed vs. context-updating). The model is fitted to two referential
experiments in German — a slider-rating study and a constrained-production study —
and a complementary factorial simulation. Bayesian leave-one-out cross-validation
quantifies how much each mechanism contributes and whether they interact.

## Repository layout

```
.
├── paper/         LaTeX manuscript, figures, paper-data CSVs, plotting scripts
├── data/          Preprocessed and raw experiment CSVs (+ stimuli table)
├── experiments/   R analyses of the slider, slider-replication, and production experiments
├── models/
│   ├── slider/        NumPyro model + inference for the slider rating data
│   └── production/    NumPyro model + inference for the production data
├── simulation/    Factorial simulation that scans speaker × semantics × scene parameters
├── analysis/      GLMM regressions and shared posterior utilities
└── Makefile       `make draft` builds paper/draft.pdf via latexmk
```

Large regenerable artifacts (MCMC `.nc` files, the full simulation CSV,
posterior-analysis output, LaTeX build directory) are gitignored.

## Building the paper

```bash
make clean-draft && make draft
```

This compiles `paper/draft.tex` via `latexmk` and copies the result to
`paper/draft.pdf`. Output is written to `paper/build/`.

## Reproducing the analyses

### 1. Python environment

The NumPyro inference code requires JAX and NumPyro. On a CPU host:

```bash
python -m venv .venv && source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

For GPU inference (recommended for the production model), replace the
`jax[cpu]` line in `requirements.txt` with the matching `jax[cuda12_pip]`
(or similar) wheel for your platform.

### 1b. R environment

The experimental data analyses and the paper's plotting + regression
scripts run in R (4.x). Install the packages used across the analyses with:

```r
install.packages(c(
  "tidyverse", "dplyr", "tidyr", "ggplot2", "purrr", "stringr",
  "lme4", "lmerTest", "emmeans", "pbkrtest", "MASS", "ordinal", "multcomp",
  "BayesFactor", "brms", "HDInterval",
  "effects", "effsize", "memisc", "sjPlot", "sjmisc", "texreg", "xtable",
  "gridExtra", "RColorBrewer", "lattice", "optimx", "trimr", "pwr"
))
```

### 2. Slider model

From `models/slider/`:

```bash
python run_inference.py --speaker_type incremental --hierarchical \
    --num_warmup 500 --num_samples 500 --num_chains 4
python posterior_analysis.py --copy-to-paper-data
```

The first command samples the posterior and writes the InferenceData to
`inference_data/*.nc`. The second extracts the summary CSVs that the paper's
plotting scripts consume.

### 3. Production model

From `models/production/`:

```bash
JAX_PLATFORMS='' XLA_FLAGS='' python run_inference.py \
    --speaker_type contextual_pcalpha_canon_parsimony \
    --hierarchical --condition-subset erdc,zrdc,brdc \
    --num_warmup 4000 --num_samples 2000 --num_chains 4
python posterior_analysis.py --copy-to-paper-data
```

The reported 2×2 (speaker × semantics) comparison on the parameter-matched best
model is driven by `run_2x2_bestmodel.sh`.

### 4. Simulation

From `simulation/`:

```bash
bash run_simulation.sh
```

This calls the two simulation scripts and writes `simulation_full_run.csv`.
The figures in the paper's Simulation section are generated by the R chunks in
`simulation/analysis.Rmd`.

### 5. Experimental data analyses

The R analyses live under `experiments/{slider,slider-replication,production}/`.
Open the respective `.Rproj` file (where present) or knit the `.Rmd` files
directly.
